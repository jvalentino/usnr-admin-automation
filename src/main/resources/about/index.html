What is this?

This application is a collection of tools written to automate various administrative activities with in the USNR. It is not officially sponsored and does not store any information, transmit any information, or host any information online. All data must be supplied locally in order to comply with PII and DISA guidelines. All tools in this application simply automate the activities of a human being on that human being's computer, using only the resources that human being has access to on that computer.

This application is a desktop application, meaning that it executes on your local machine. It is not a website or web application and stores nothing online.

How?

This tool suite is a throw back to the era of the desktop application. While it would make more sense to build yet-another-web-application using the latest in cloud technologies, there are rules and regulations around storing and operating these systems when dealing with FOUO and PII data. A desktop application runs locally, stores locally, and in this case must have data supplied to it locally. Most importantly is solves a current need now, and can be easily modified.

Change History

1.24 - Mar 13, 2015 - No more having to convert XLS to XLSX

  • The Report Merger now has LibreOffice integration, which will allow you to directly open XLS files instead of having to first convert them to XLSX.
  • LibreOffice is a free office style application, which can be downloaded from libreoffice.org
  • If you install it in the standard location this application will automatically detect it, otherwise you will have to tell it where you installed LibreOffice.

1.23 - Mar 11, 2015 - Being able to ignore Courses

  • Manual Inputs.xlsx - The Settings sheet now accepts rules that can be used to ignore courses.
  • This is useful for those units that do IC IAA V12 instead of IAA V12
  • To ignore a course use Action: IGNORE COURSE Value: IC IAA V12

1.22 - Mar 10, 2015 - Tracking Historical Data

  • Manual Inputs.xlsx - After generating report there a Historical tab will be added to the workbook for the current date.
  • Historical data comes from keeping the summary percentages for every tracked item for the current date.
  • Every time reports are generated a new column for the current date will be added to the Historical tab.
  • This was done so that you can see training percentages over time for whatever period you want to track.
  • You can delete any of the date columns within the Historical tab to remove them, and you can also just delete the Historical tab to reset history.

1.21 - Mar 6, 2015 - Navy Websites

  • Added a list of Navy websites and an option in the left menu called Navy Websites

1.20 - Mar 6, 2015 - Mass Communication

  • The SMS Sender has been changed to the Mass Message Sender
  • It is capable of sending both text messages and emails

1.19 - Mar 1, 2015

  • Report Merger - After generating reports there is now a button to Send Training Notifications. This will allow you to email or SMS every selected individual with the eLearning and GMT Category II training that they need to do online.

1.18 - Feb 28, 2015
